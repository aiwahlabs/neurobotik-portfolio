Final Interview Responses

1. Reporting ownership: What's the most relevant monthly ops report you've owned end-to-end? What KPIs did you track and what decisions did it drive?

At a real estate tech startup, I served as the Operations and Tech Lead, reporting directly to the CEO and managing a team of 40+ agents. I owned the end-to-end delivery of the Customer Success Yield Report, which synthesized data from more than 10,000 monthly interactions into strategic insights.

I tracked Cost-per-Resolution, Agent Utilization Rate, and Intent-to-Escalation Ratio. By analyzing this data, I identified that 45% of our overhead was spent on low-complexity inquiries (pricing/booking). I presented a business case to the leadership team linking this inefficiency to our inability to scale during peak seasons. This drove the decision to implement our first AI voice triage system, reducing operational costs by 32% and allowing me to reallocate my team to high-value account management, directly impacting our quarterly retention goals.

2. Data discipline: How do you ensure the live dashboard numbers match your monthly Excel report (single source of truth, metric definitions, versioning)?

Everything flows from a Single Source of Truth - typically a locked Google Sheet or database table that all automations write to. Dashboards (Looker Studio) and Excel reports both pull from this same source, never from each other.

For versioning, I use date-stamped snapshots and maintain a metric definitions doc so "Conversion Rate" means the same thing everywhere. In n8n, I add validation nodes to compare expected vs actual row counts - this catches silent failures before they corrupt the monthly report.

3. Excel + analysis depth: Talk me through building a refreshable Excel report from exports: cleaning, de-duping, pivoting, and producing a one-page exec summary.

I use Power Query to connect to raw CSV/JSON exports. The workflow is: 1. Auto-clean (normalize date formats, trim whitespace), 2. Remove duplicates using unique SessionIDs, 3. Merge with reference tables (agent names, intent categories).

The output feeds into a Pivot Table grouped by week and intent type. For the exec summary, I use conditional formatting to highlight red/green on KPIs like Deflection Rate and Avg Response Time - one page, no scrolling, with a 30-second read time target, decision-ready.

4. Root-cause analysis: Escalations jump 30% week-over-week. Talk me through your first five checks and how you'd isolate the cause.

I would approach this 30% jump using a systematic Segment-Timeline-Hypothesis framework within the first 4 hours of detection:

  1. Segment: I first isolate the spike. Is it concentrated in a specific User Intent (e.g., Cancellation requests), a Geographic Region, or a specific Platform (e.g., WhatsApp vs. Web)?
  2. Timeline: I check if this was a Sudden Spike (suggesting an infrastructure failure or a marketing blast) vs. a Gradual Drift (suggesting model decay or knowledge base obsolescence).
  3. Input Integrity: I audit the last 50 escalated logs. Are we seeing "Garbage In" where the model is receiving malformed user data?
  4. Tool Benchmarks: I check the latency and error rates for our integrated tools (e.g., Project Lookup API). A slow tool often triggers a fallback escalation.
  5. Hypothesis Testing: Based on the above, I form a hypothesis (e.g., "The new marketing campaign is attracting users from a region we don't yet have KB coverage for") and validate it by sampling the specific data segment.

This allows me to move from "reaction" to "resolution" by pinpointing exactly where the logic or data coverage broke down.

5. Automation monitoring: When you automate workflows/agents, how do you monitor failures and quality drift over time? What do you track weekly?

I build a robust Instrumentation Chain for every automation to ensure zero silent failures. My monitoring stack typically follows this flow:

  • n8n Error Handling: Every workflow has an Error Trigger node that alerts me instantly via Slack/Discord if a tool or API fails.
  • Data Warehouse (BigQuery): All interaction logs (User query, AI response, Metadata) are automatically mirrored to BigQuery.
  • Visualization (Looker): I build a weekly dashboard tracking Fallback Rate and Tool Success Latency.

Weekly, I conduct a "Ground Truth" Calibration: I take a random 5% sample of AI outputs and score them against a human-verified "perfect" response. This allows us to track Semantic Drift over time‚Äîensuring the AI isn't just answering correctly, but is maintaining the high-quality tone Neurobotik expects.

6. Incident scenario: A client says the assistant is 'making things up' in edge cases. What steps do you take in 48 hours to diagnose and fix?

Diagnosis (0-12h): I pull the specific session ID from our Global Interaction Log in n8n. I perform a "Ground Truth" comparison: I overlay the user's query against our Knowledge Base to identify if it's a Retrieval Failure (the AI didn't find the info) or a Generation Failure (it found it but hallucinated the answer).

Isolation (12-24h): I use log sampling to check if this "hallucination" is isolated or systemic across similar edge cases. If systemic, I immediately lower the model temperature and add a Negative Constraint to the system prompt (e.g., "Do not infer pricing if not explicitly stated").

Fix & Verification (24-48h): I update the RAG ranking logic or the prompt and run it through our historical test suite to ensure no regressions. I provide the client with a Post-Incident Report detailing the root cause and the permanent guardrail implemented.

7. Dashboard maintenance: If we start with Looker Studio + Google Sheets and later move to BigQuery, what's your approach to data modeling and ongoing upkeep?

For the migration: I use n8n to handle ETL, writing to BigQuery instead of Sheets directly. Key is flattening nested JSON early in the pipeline so SQL queries stay clean and performant.

For ongoing upkeep: I document schema changes in a changelog, and Looker dashboards point to views (not raw tables) - so I can modify the underlying structure without breaking live reports.
